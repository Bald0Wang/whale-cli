# Whale CLI 垂直学习智能体：模块设计

## 目标

把已有的通用 CLI Agent 变成 Datawhale 学习者可直接使用的陪学系统。学习者只需描述基础、目标和可投入时间；系统负责检索资料、建立学习状态、给出下一步、安排复习，并把成果沉淀为可回顾的档案。

本设计不把“记忆”“知识图谱”或“自动学习”包装成黑盒。每一个结论都要能回到用户填写的数据、知识库资料或学习记录。

## 学习者的一次完整旅程

```text
填写学习档案
  -> Datawhale 知识库检索与项目建议
  -> 建立概念节点和前置关系
  -> 生成当前可执行的学习路线
  -> 完成练习、登记项目证据
  -> 按间隔复习回顾概念
  -> 导出学习档案、提出社区补充建议
```

## 代码模块与数据契约

所有学习者数据都保存在当前工作区的 `.whale_cli/learning/state.json`。文件是用户可读的 JSON，不上传到模型服务；模型只在用户请求时通过工具读取必要的摘要。

| 模块 | 核心职责 | 输入 | 输出 | 教程 |
|---|---|---|---|---|
| `learning/profile.py` | 保存学习者入口信息 | 基础、目标、周时长、偏好 | `LearnerProfile` | 22 |
| `learning/knowledge.py` | 维护概念与双向链接 | 节点、关系、掌握度 | 节点详情、正向/反向链接 | 23 |
| `learning/roadmap.py` | 根据依赖和时间生成小步任务 | 档案、知识节点、里程碑 | `RoadmapItem` 列表 | 24 |
| `learning/review.py` | Ebbinghaus 风格的间隔复习 | 概念、回忆评分 | 下次复习日期、到期卡片 | 25 |
| `learning/projects.py` | 生成项目陪学计划，受控克隆项目 | 项目 URL、目标目录 | 本地项目记录或明确错误 | 26 |
| `learning/portfolio.py` | 汇总证据、报告和贡献草稿 | 完成记录、笔记、路线 | Markdown 学习档案 | 27 |
| `tools/learning/` | 为 Agent 暴露稳定工具接口 | 工具参数 | 标准 Tool 返回值 | 22-27 |

`LearningStore` 是唯一写入点，负责原子写入。各领域服务不直接读写文件，因此后续替换 SQLite、远端同步或 WebUI 表单时，不会改变路线、复习和图谱规则。

## 关键原则

- **先询问，后规划**：没有学习档案时，路线工具只返回缺失字段，不猜测用户基础。
- **双链不是双份数据**：每条边只存一次；查询节点时同时计算出链入和链出，避免数据漂移。
- **计划可以重排**：路线项保存状态，不把一次模型建议当成永久课程表。
- **复习属于学习者**：评分和日期在本地，默认不调用模型。
- **克隆必须审批**：项目陪学可以组织 `git clone`，但它是写入工作区和联网动作，必须走已有审批层。
- **贡献草稿不自动提交**：系统只生成可审阅的 Markdown 建议；用户决定是否创建 PR。

## 分期顺序

第 21 章已经完成“资料检索 + 路线建议”。22 到 25 章形成最小陪学闭环，26 到 27 章把练习落到项目和可展示成果。知识库同步、语义检索、远端账户、自动 PR 与课程平台对接均留在后续扩展，不能在当前教程中描述为已实现。
