# 24. 动态学习路线：下一步只做一件事

本章导航：

- 新增机制：基于档案、掌握度和前置关系的本地路线生成，并自动归档为 JSON。
- 正式入口：`learning/roadmap.py`、`tools/learning/learning_tools.py`。
- 验证方式：未完成 Python 基础时，不解锁 Agent Loop；每次生成都会留下可在 WebUI 查看的路线快照。
- 本章不展开：自动报名、课程平台进度同步和强制日程安排。

路线不是一次生成四十节课的日历，而是每次只给学习者一个当前能完成的下一步。它必须解释“为什么是这一步”，并允许重新生成。

## 24.1 路线规则

`RoadmapPlanner` 读取三类本地事实：

- 档案中的周时长，决定单项建议投入多少时间。
- 节点 `mastery`，优先处理掌握度低的概念。
- `prerequisite` 边，阻止尚未准备好的高级概念进入路线。

```text
档案完整？ -> 否：指出缺失字段
           -> 是：找掌握度较低的节点
                 -> 检查前置节点
                 -> 输出 1 到 N 个可执行路线项
```

每个路线项有 `id`、周次、概念、预计小时、状态、`reason` 和 `subtasks`。`reason` 是给学习者看的解释，不是内部调试信息。`subtasks` 把一个周目标拆为三个能独立关闭的学习步骤：先明确概念，再完成最小练习，最后沉淀复盘记录。

这层拆分很重要。`掌握 Agent Loop` 是方向，不是一个可以在三小时后诚实打勾的动作；而“写下核心概念”“让一个最小循环跑起来”“记录本次卡点”都有明确的完成证据。每个子任务按 25% / 55% / 20% 分配父任务预算，帮助学习者从启动、动手到收尾，而不是面对一块过大的待办。

## 24.2 调用方式

先只生成草案，不写入本地路线：

```json
{"action": "preview", "weeks": 4}
```

Whale 应展示每周任务、时间预算和原因，再问学习者“要生成这条路线吗？”。学习者明确回答确认后，才调用：

```json
{"action": "generate", "weeks": 4}
```

`generate` 会写入 JSON，因此在 Safe 模式还会出现一次审批。预览不会改变 `state.json`、路线历史或 WebUI 页面。

完成一个小项也必须由学习者触发。WebUI 默认在子任务旁逐条勾选；CLI 中说“我已完成 Agent Loop”仍兼容原有习惯，会将这个周目标下剩余步骤一起标为完成。WebUI 勾选后才调用：

```json
{"action": "complete", "roadmap_id": "roadmap-python-基础-practice"}
```

第一个子任务完成时，父目标状态变为 `in_progress`；三个子任务都完成时，父目标才变为 `done`。完成状态会保存，但不会自动把概念掌握度改成 5。学习者还应该通过练习、复习或自评更新地图，这能避免“看完即掌握”的假进度。

当学习者说“我做完了”时，Agent 应先读取当前路线，而不是马上生成新路线：

```json
{"action": "status"}
```

从返回的 `items` 找到原有 `roadmap_id` 后再调用 `complete`。只有完成状态写入后，才根据新的知识地图决定是否生成路线；否则新生成的路线会覆盖当前路线，完成标记可能落到错误的任务上。

## 24.3 每次生成都会留下路线快照

过去的 `roadmap` 只有“当前路线”：再次生成时会被替换。现在 `RoadmapPlanner.generate()` 在更新当前状态的同时，会把当时的路线保存为独立 JSON：

```text
.whale_cli/learning/
├── state.json                  # 当前路线和学习状态
└── roadmaps/
    ├── route-20260713T....json # 第一次生成的快照
    └── route-20260714T....json # 后续重新规划的快照
```

一份快照包含生成时间、当时的学习者档案、路线项、子任务和汇总数据。它不保存模型隐藏推理，也不上传到服务器：

```json
{
  "version": 1,
  "id": "route-20260713T090000000000Z-a1b2c3d4",
  "generated_at": "2026-07-13T09:00:00+00:00",
  "profile": {"goal": "完成 Agent 项目", "weekly_hours": 6},
  "items": [{
    "week": 1,
    "title": "掌握 Agent Loop",
    "status": "in_progress",
    "subtasks": [
      {"id": "roadmap-agent-loop-understand", "status": "done"},
      {"id": "roadmap-agent-loop-practice", "status": "todo"},
      {"id": "roadmap-agent-loop-reflect", "status": "todo"}
    ]
  }],
  "summary": {"item_count": 1, "done_count": 0, "subtask_count": 3, "subtask_done_count": 1, "estimated_hours": 3.6}
}
```

完成 `LearningRoadmap({"action":"complete"})` 后，当前路线对应 JSON 的 `items`、父任务 `done_count`、子任务进度和 `updated_at` 会同步更新。早期没有 `subtasks` 的路线在读取时会自动补齐三步，不会因为升级而无法查看或勾选。重新生成一条路线会新建快照，而不是覆盖之前的方向，方便回顾“当时为什么这样安排”。

## 24.4 在 WebUI 中快速找到方向

WebUI 左侧“学习路线”读取 `.whale_cli/learning/roadmaps/`，约每 2.5 秒静默刷新。点击“预览新路线”先看到草案，只有点击“确认生成路线”才会写入 JSON。页面不直接堆出 JSON，而是套用固定模板：

- 左侧是按生成时间排序的路线档案，可切换查看旧计划。
- 顶部显示目标、当前基础、周时长和完成进度。
- “下一步”显示第一条未完成的子任务及其具体动作，帮助学习者先动起来。
- 周次时间线保留任务、时间预算和安排理由；每个周任务下显示三个子任务以及各自的预计时间。仅当前路线的待办子任务可以勾选关闭，历史路线始终只读。

点击“通过对话规划”会回到对话框并预填预览任务。Whale 先检查 `LearnerProfile` 与 `KnowledgeMap`，信息完整后调用 `LearningRoadmap(preview)` 并等待确认。路线保存后，JSON 会自动出现在这个页面；无需手动上传、导入或刷新浏览器。

## 24.5 难与易的处理

遇到太难的项目：把它需要的概念节点和前置边补进图谱，再生成路线。遇到太简单的内容：把节点掌握度调高，路线会更倾向项目实践和复盘。这个规则比让模型随口判断难度更可检查。

Datawhale 项目推荐仍应先走第 21 章的知识库子代理；路线模块只负责把已知概念安排成下一步。

## 24.6 实用案例：小陈本周到底先做什么

第 22、23 章已经保存了小陈的档案和关系：`Python 基础`掌握度为 4，`Agent Loop`掌握度为 1，且前者是后者的前置条件。小陈输入：

```text
根据我的学习档案和知识地图，先预览四周路线；告诉我本周第一件要做的事和理由，并问我是否确认生成。
```

Whale 调用：

```json
{"action":"preview","weeks":4}
```

当前实现会将 `Agent Loop` 放入草案，因为它掌握度较低、且 `Python 基础`已经达到可用水平。输出中的 `reason` 会说明“当前掌握度较低，且已满足已记录的前置条件”；`estimated_hours` 会根据档案中的 6 小时周预算计算。小陈确认后，Whale 才调用 `{"action":"generate","weeks":4}`。完成后，小陈可以说“我已完成 Agent Loop”，也可以在 WebUI 中先完成最小练习，再勾选复盘：

```json
{"action":"complete","roadmap_id":"roadmap-agent-loop-practice"}
```

这不会假装他已经完全掌握 Agent Loop，而是把“做完一个安排”与“真正记住一个概念”分开，下一章用复习记录后者。

小陈打开 WebUI 的“学习路线”，先在草案中看到“完成 Agent 项目”，确认后才形成第一条档案。右侧的“下一步”会先显示“明确 Agent Loop 的核心概念”，完成后才轮到“完成 Agent Loop 的最小练习”，并分别标出时间预算和完成证据。两周后，小陈补齐了新的前置知识并重新预览路线：旧路线仍保留在左侧，新路线在他确认后才会作为新的 JSON 快照排在最上面。这样他能看见自己的方向如何变化，而不是只剩一份被覆盖的计划。

## 24.7 验证

```bash
./.venv/bin/python -m pytest tests/test_learning.py tests/test_webui_settings.py -q
```

测试构造两个掌握度为 0 的节点，并声明“Python 基础”是“Agent Loop”的前置条件。生成结果只包含 Python 基础，证明路线没有跳关；另有测试确认每条周目标会生成三个子任务，完成其中一步会使父任务进入进行中状态，完成全部步骤后才关闭父任务并更新 JSON 快照，WebUI API 能读取这些档案。

## 当前边界

- 当前是透明规则，不是黑盒推荐算法。
- 路线不会自动读取在线课程完成度。
- 它不承诺固定周期内一定学会；周时长只是计划预算。
- JSON 归档只保存在当前工作区；删除工作区或手动删除 `roadmaps/` 会失去对应的路线历史。
