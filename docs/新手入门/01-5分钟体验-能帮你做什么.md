# 01. 5 分钟体验：你的第一个 CLI Agent 任务

本章导航：

- 新增机制：启动 CLI、发送普通消息，并观察一次只读工具调用。
- 正式入口：`src/whale_cli/ui/shell/main.py`。
- 验证方式：运行 `whale-cli` 后完成一次对话和一次仓库探索。
- 本章不展开：模型请求格式、会话落盘和工具分发会在 02 至 06 章拆开讲。

> 目标：在不改源码的前提下，完成一次对话和一次只读工具调用。先确认它能工作，原理留到后面章节。

| 检查项 | 命令/操作 | 预期结果 |

## 1. 启动 Whale CLI（30 秒）

```bash
whale-cli

# 在仓库中开发时，也可以通过模块方式启动
PYTHONPATH=src python -m whale_cli.ui.shell.main
```

PowerShell 中把第二条命令写成：

```powershell
$env:PYTHONPATH = "src"
python -m whale_cli.ui.shell.main
```

**成功标志：** 看到下面的欢迎语，光标停在 `User>` 后面等待输入

你会看到 Whale CLI 的 ASCII logo，以及：

```text
Whale CLI - your terminal coding sidekick
Type '/help' for commands. Type 'exit' or 'quit' to leave.
User> _
```

## 2. 第一条命令：让它自我介绍（1 分钟）

直接复制粘贴这句话：

```
你好，请介绍一下你自己，并列出你能使用的工具。
```

**你应该看到：**
1. 先出现 `[Thinking...]` - 表示正在思考
2. 然后出现 Whale 的回复。具体措辞、工具顺序和工具选择会随模型、操作系统与当前配置变化。

```
Whale: 我是 Whale CLI 助手。我可以读取文件、搜索内容、执行命令，并在确认后修改文件。
```

检查点 1：看到一条正常的模型回复，说明 CLI 与模型的基本对话链路可用。

---

## 3. 第一条工具调用：探索当前目录（1 分钟）

复制粘贴：

```
请用 Bash 工具查看当前目录下有哪些文件和文件夹
```

**你应该看到：**

```
[Thinking...]
[Tool Call] Bash({"command": "ls -la"})
[Tool Result] {"stdout": "total 128\ndrwxr-xr-x  6 user..."}...

Whale: 当前目录包含以下内容：
- docs/ - 文档目录
- src/whale_cli/ - 内部 Python 包目录
- README.md - 项目说明
...
```

检查点 2：日志中出现 Tool Call 和 Tool Result，说明模型、Toolset 和工具回填链路可用。

## 4. 可选体验：创建文件

这一步会触发写文件审批。请只在临时目录或你确认可写的项目目录中运行。

复制粘贴：

```
请在当前目录创建一个名为 hello.txt 的文件，内容为 "Hello from Whale CLI!"，然后读取确认内容正确。
```

**你应该看到：**

```
[Thinking...]
[Tool Call] WriteFile({"path": "hello.txt", "content": "Hello from Whale CLI!"})
[Tool Result] {"stdout": "File written successfully", ...}...
[Thinking...]
[Tool Call] Bash({"command": "cat hello.txt"})
[Tool Result] {"stdout": "Hello from Whale CLI!", ...}...

Whale: 已成功创建 hello.txt 文件，内容为 "Hello from Whale CLI!"，读取确认无误。
```

去你的文件管理器查看，应该能看到 `hello.txt` 文件！

检查点 3：确认审批后文件被创建，说明写入工具和审批链路可用。

联网工具依赖当前网络和搜索服务。它不是本章的必过项。

检查点 4：出现搜索工具调用并得到可读结果，说明网络工具在当前环境可用。

## 6. 观察重点：Agent 的工作方式

完成上面 4 个任务后，你应该已经注意到 Whale CLI 的工作模式：

![ReAct Agent Loop 闭环](images/react-agent-loop.svg)

```
你提需求 → [Thinking...] → 可能调用工具 → 观察结果 → 给你回答
```

**关键观察点：**

| 现象 | 含义 |
|-----|------|
| `[Thinking...]` | Agent 正在分析你的需求 |
| `[Tool Call] xxx` | Agent 决定使用工具来解决问题 |
| `[Tool Result] xxx` | 工具返回的结果 |
| `Whale: xxx` | Agent 基于观察给出最终回答 |

这就是 **ReAct 循环**：思考 → 行动 → 观察 → 回答

## 你现在应该建立的直觉

1. **Whale CLI 不只是聊天** - 它会主动使用工具来完成任务
2. **过程可见** - 你能看到它什么时候在思考、什么时候在调用工具
3. **结果可验证** - 文件真的在磁盘上、搜索真的能联网

这些特性让 Whale CLI 成为一个"能干活的工程搭档"，而不只是一个"会说话的聊天机器人"。

## 遇到问题先看这里

| 现象 | 先检查什么 |
|---|---|
| 找不到 `whale-cli` | 在项目根目录执行 `pip install -e .`，或使用上面的模块启动命令 |
| 提示缺少 API Key | 设置 `STEP_API_KEY`，或按 03 章配置 `~/.whale/config.json` |
| 没有 Tool Call | 换成第 3 节的明确指令；模型不一定会为闲聊调用工具 |
| 写文件被拒绝 | 这是正常的审批结果；重新发起任务并在提示时确认 |
| 联网搜索失败 | 先跳过本节，不影响后续离线教程 |

## 本章模块化代码

本章只要求你“跑起来”，所以代码重点看入口文件和命令注册。

### 1. 包入口命令

文件：`pyproject.toml`

```toml
[project.scripts]
whale-cli = "whale_cli.ui.shell.main:main"
```

安装后执行 `whale-cli`，实际就是调用 `src/whale_cli/ui/shell/main.py` 里的 `main()`。

### 2. 启动横幅和本地数据目录

文件：`src/whale_cli/ui/shell/main.py`

def _get_store() -> SessionStore:
    base_dir = (
        os.environ.get("WHALE_CLI_HOME")
        or os.path.join(os.getcwd(), ".whale_cli")

这两段对应你启动时看到的 logo，以及项目目录下自动出现的 `.whale_cli/` 数据目录。

### 3. 启动时创建或恢复会话

```python
def _start_new_session(store: SessionStore) -> tuple[Soul, str]:
    session_id = store.create_session(title="")
    agent = Soul(session_store=store, session_id=session_id)
    return agent, session_id


def _restore_latest_session(store: SessionStore):
    session_id = store.get_latest_session_id()
    if not session_id:
        return None, None
    messages = store.load_messages(session_id=session_id)
    if not messages:
        return None, None
    agent = Soul(session_store=store, session_id=session_id, initial_messages=messages)
    return agent, session_id
```

读到这里你就能理解：Whale CLI 不是一次性脚本，它从第一章就已经有“会话”这个概念。
