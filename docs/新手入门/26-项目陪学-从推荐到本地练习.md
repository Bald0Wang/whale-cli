# 26. 项目陪学：从推荐到本地练习

本章导航：

- 新增机制：带学习定位的项目练习计划、仓库 URL 校验与审批后的本地克隆。
- 正式入口：`learning/projects.py`、`tools/learning/learning_tools.py`。
- 验证方式：拒绝非仓库 URL 和工作区外目录；克隆工具声明审批动作。
- 本章不展开：替学习者完成课程、自动运行不可信项目或自动提交代码。

资料推荐之后，学习者还需要一个能动手的项目。但“自动帮我拉项目”必须同时考虑体验和安全：系统可以准备好练习路径，真正写入磁盘和访问 Git 平台时仍由用户确认。

## 26.1 两阶段而不是一键克隆

第一阶段创建计划，不联网、不克隆。计划不是“下载待办”，而是先让学习者看懂这个项目为什么值得做、开始前缺什么、完成后能拿出什么证据：

```json
{
  "title": "Agent Learning Hub 练习",
  "url": "https://github.com/datawhalechina/Agent-Learning-Hub.git",
  "goal": "读懂一个工具调用示例，并做一次小改动",
  "learning_value": "把 Agent Loop 从抽象概念变成一次可观察的项目流程。",
  "prerequisites": ["Python 基础", "Agent Loop"],
  "outcomes": ["解释工具结果如何回填", "完成一次最小改动并验证"],
  "first_action": "只读 README，标出示例入口、输入和预期输出。"
}
```

`LearningProjectPlan` 会保存项目标题、URL、目标、建议目录和三个带证据的里程碑：理解项目定位、验证最小闭环、完成一次有解释的改动。每一步都写明目的和应留下的证据，因此学习者不会把“克隆成功”误判为“项目学会了”。

第二阶段才克隆：

```json
{
  "url": "https://github.com/datawhalechina/Agent-Learning-Hub.git",
  "directory": "projects/agent-learning-hub"
}
```

`CloneLearningProject` 具有 `approval_action = "clone learning project"`。在 Safe 模式下，WebUI 会请求同意；YOLO 模式也会在运行轨迹中留下记录。

## 26.2 安全边界

工具只接受直接的 GitHub 或 Gitee 仓库 URL，不接受任意网页。目标目录必须：

- 位于当前工作区内。
- 尚不存在。
- 不能使用 `..`、绝对路径或符号链接逃离工作区。

实际执行使用参数数组而不是 `shell=True`：

```python
subprocess.run(["git", "clone", "--depth", "1", url, target])
```

这不是完整沙箱。克隆后的项目代码仍然不应在未经理解和审批的情况下运行。

## 26.3 困难与简单的项目

项目太难时，不要硬做：先把“缺哪个前置、为什么缺它”写入第 23 章的知识图谱，再回到第 24 章生成补充路线。项目计划中的 `prerequisites` 是学习前检查项，不是系统替你猜测的掌握度。

项目太简单时，也不要只打勾结束：把 `outcomes` 提高到“解释一个设计选择”“增加一个测试”或“写一份可复现报告”，然后用第 27 章的学习档案记录实际产出。项目的难度应该由下一步需要建立的能力决定，而不是由仓库 star 数决定。

## 26.4 实用案例：把第 21 章推荐的仓库带进工作区

小陈从第 21 章的输出中选定一个 Datawhale 仓库。先让 Whale 写计划，而不是立即下载：

```text
为第 21 章推荐的 Agent 学习仓库建立练习计划。请说明它为什么适合我当前目标、
我需要先检查哪些前置知识、学完能做什么，并给一个只读 README 的第一步。先不要克隆。
```

Whale 应调用 `LearningProjectPlan`，其中 `url` 必须是第 21 章输出里已经核对过的直接 GitHub 或 Gitee 仓库 URL。计划返回后，小陈确认仓库和目录，例如：

```text
确认把这个项目克隆到 projects/agent-learning-hub；请继续。
```

这时才调用：

```json
{
  "url":"<第21章已核对的仓库 URL>",
  "directory":"projects/agent-learning-hub"
}
```

在 Safe 模式中，WebUI 会显示“clone learning project”审批。小陈应确认目录在工作区内、目标目录不存在，且只在阅读 README 后再决定是否运行项目命令。这样“陪学”减少了准备工作，却不会绕过安全判断。

## 26.5 验证

```bash
./.venv/bin/python -m pytest tests/test_learning.py -q
```

测试模拟 `git clone`，验证 URL 白名单、工作区边界、克隆后的 `ready` 状态以及带前置、产出和证据的项目计划；不会真实联网克隆仓库。

## 当前边界

- 当前不会自动执行项目安装、脚本或测试。
- 当前不读取仓库 issue，也不自动创建 PR。
- 项目推荐事实仍来自第 21 章的本地 Datawhale 知识库；本章只负责练习落地。

下一章把做过的练习和反思沉淀成学习者自己的作品档案。
