# 22. 学习者档案：先知道要帮谁

本章导航：

- 新增机制：本地学习者档案与缺失字段检查。
- 正式入口：`learning/profile.py`、`tools/learning/learning_tools.py`。
- 验证方式：保存基础、目标和每周时间，再读取档案。
- 本章不展开：账号登录、跨设备同步和行为画像。

“给我一份学习路线”如果没有上下文，通常只会得到一份看似完整、其实不适合任何人的清单。学习系统的第一件事不是猜用户，而是用极少的信息建立一个明确入口。

## 22.1 最小档案

Whale CLI 只要求三项：

- `level`：当前基础，例如“Python 初学者”。
- `goal`：想完成什么，例如“完成 Agent 项目”。
- `weekly_hours`：每周真实能投入的小时数。

`topics` 和 `learning_style` 是可选项。它们帮助调整表述和选题，但不应成为进入学习系统的门槛。

```text
学习者填写档案
  -> LearnerProfileService.update()
  -> .whale_cli/learning/state.json
  -> 路线、复习、档案模块共用同一份事实
```

## 22.2 为什么要本地保存

学习档案属于学习者，不是模型的“长期记忆”。当前版本写入工作区：

```text
.whale_cli/learning/state.json
```

模型只会在工具调用时看到必要字段；换模型、重新打开会话或离线查看时，档案仍然可读。`LearningStore` 是唯一写入点，后续改成 SQLite 或同步服务时，不需要修改路线算法。

## 22.3 工具调用

```json
{
  "action": "update",
  "level": "Python 初学者",
  "goal": "四周完成一个 Agent 小项目",
  "weekly_hours": 6,
  "topics": ["Python", "Agent"],
  "learning_style": "先做小练习再读原理"
}
```

对应工具是 `LearnerProfile`。查询时只需：

```json
{"action": "show"}
```

若档案不完整，`missing_fields()` 会直接返回缺少项；路线模块会停止并提示补齐，而不是伪装成个性化建议。

## 22.4 实用案例：第一次进入陪学模式

小陈拿到第 21 章的推荐后，不想每次重新解释自己的情况。他在对话框输入：

```text
请帮我开始陪学：我是 Python 初学者，目标是四周完成 Agent 小项目，
每周可以投入 6 小时。我偏好先做小练习再读原理，主题是 Python 和 Agent。
```

Whale 应调用 `LearnerProfile`：

```json
{
  "action": "update",
  "level": "Python 初学者",
  "goal": "四周完成 Agent 小项目",
  "weekly_hours": 6,
  "topics": ["Python", "Agent"],
  "learning_style": "先做小练习再读原理"
}
```

调用后，小陈的工作区会多出 `.whale_cli/learning/state.json`。他可以追问“我的学习档案缺什么”，Whale 会用 `{"action":"show"}` 返回档案和 `missing_fields`；这比让模型从旧聊天中猜测基础可靠。

## 22.5 验证

```bash
./.venv/bin/python -m pytest tests/test_learning.py -q
```

测试验证档案会落到本地 JSON，且“基础、目标、周时长”缺一不可。

## 当前边界

- 没有登录、云同步或隐式行为追踪。
- 不会根据聊天内容偷偷修改学习者基础。
- 档案是当前工作区级别；多人共用目录时，应使用不同工作区。
