# 25. 间隔复习：让学过的内容留下来

本章导航：

- 新增机制：本地回忆评分、聊天记录检索与逐级间隔复习表。
- 正式入口：`learning/review.py`、`tools/learning/learning_tools.py`。
- 验证方式：高分回忆拉长间隔，低分回忆回到短间隔；已聊过的知识点能进入本地复习表。
- 本章不展开：推送通知、医学或教育学效果承诺。

学习完成不等于长期记住。这里实现的是一个轻量、可解释的复习排程器：让学习者自己评价回忆质量，系统再决定下一次回顾的日期。

## 25.1 评分与间隔

对某个知识节点给 0 到 5 分：

- 0 到 2：没能稳定回忆，回到第 1 阶段。
- 3 到 5：通过本次回忆，向下一阶段移动。

阶段间隔是：`1, 3, 7, 14, 30, 60` 天。它是一套可读的 Ebbinghaus 风格近似，不是对所有人都最优的科学处方。

```text
评分 5 -> stage 1 -> 3 天后复习
评分 1 -> stage 0 -> 1 天后复习
```

## 25.2 调用方式

先取到期卡片：

```json
{"action": "due"}
```

学习者尝试不看笔记解释概念后，再写入评分：

```json
{"action": "rate", "concept_id": "agent-loop", "rating": 4}
```

评分写在 `.whale_cli/learning/state.json`，包括尝试次数、当前阶段、上次复习和下次到期日。模型不应仅凭一段聊天文字自动替用户打分。

## 25.3 从聊天记录建立复习表

学习者通常不会记得每个知识点是什么时候聊过的。`LearningReview` 新增了两个动作：

```json
{"action": "sync"}
```

它读取当前工作区 `.whale_cli/sessions/` 下的本地 JSONL 会话，逐条检查用户和助手消息。扫描器只匹配已经存在于 `KnowledgeMap` 中的概念标题或 ID。例如，知识图谱已有 `Agent Loop`，聊天里出现“解释 Agent Loop”，它就会被加入待复习表。

```json
{"action": "schedule"}
```

它返回当前表，包括到期卡、后续卡、上次扫描信息和聊天匹配次数。表也会写入：

```text
.whale_cli/learning/review-schedule.json
```

同时会生成一份给人阅读和手动勾选的 Markdown 反馈：

```text
.whale_cli/learning/review-feedback.md
```

在 CLI 中可以主动要这份反馈：

```json
{"action": "feedback"}
```

它会列出今天的复习清单、每个概念的回忆提示和一句可直接发给 Whale 的评分指令。Markdown 里的勾选框方便整理笔记；真正完成一次复习仍要由学习者自己给出 0 到 5 的回忆评分。

新卡默认当天到期，原因是“聊天提到过”只能证明它值得回忆，不能证明已经掌握。扫描不会从语言风格、聊天长度或模型回答质量推断分数，也不会改写节点掌握度。

## 25.4 WebUI 与定期检索

WebUI 左侧有单独的“间隔复习”栏目，会读取这张本地表。CLI 或 WebUI 启动时都会在后台按当天日期检查一次；页面打开后每分钟查询一次，但后端每天最多自动扫描一次聊天记录，避免重复遍历会话文件。点击“检索聊天记录”可手动立即扫描；点击 0 到 5 的评分后，系统按本章的间隔规则计算下次日期。

页面包含四列：概念、下次复习、发现来源和回忆评分；评分是这张表真正的“完成”动作。顶部会显示待复习数量、后续安排数量、已检索消息数和表的 JSON 路径，并可直接查看同一份 Markdown 反馈。它只监听本地工作区，不发送聊天记录到第三方。

选择表中的一个概念，会展开“本地复习资料”：知识地图中为该节点保存的摘要、前置或关联概念，以及最多 6 条命中过该概念的历史对话片段。资料默认不展开，学习者可以先独立回忆，再按需查看。片段只会取你的记录和 Whale 的最终回复，不展示内部工具 JSON。这个过程不调用模型；它只是读取本地 `state.json` 和 JSONL 会话。CLI 也可读取同样的资料：

```json
{"action": "detail", "concept_id": "agent-loop"}
```

每个词条详情还会显示一张 Canvas 绘制的“艾宾浩斯遗忘曲线辅助图”。首次评分前，它只显示短间隔的基线示意；学习者提交过 0 到 5 的评分后，曲线会使用该次评分对应的复习间隔、上次复习日期和下次到期日，显示当前节奏下的估计保持率与“今天”位置。它不是记忆能力测量，也不能替代真实的回忆练习；它的作用是让学习者直观看到为什么此刻该复习、下一次将安排在何时。

当用户在对话中说“根据聊天记录生成复习表”或“今天该复习什么”时，Whale 应先调用：

```json
{"action": "sync"}
```

再调用：

```json
{"action": "schedule"}
```

这让 LLM 负责解释和陪练，复习事实仍由可检查的本地工具产生。

## 25.5 与路线如何配合

路线负责“学什么”，复习负责“回顾什么”。二者不互相覆盖：完成路线项不等于通过回忆检查；复习低分也不等于失败，而是提醒路线可能需要拆得更小或补一个前置概念。

## 25.6 实用案例：用两分钟检查 Agent Loop 是否真的留下来

小陈完成了本周的工具调用练习。之后他和 Whale 聊过“Agent Loop 如何把工具结果回填”。打开 WebUI 的“间隔复习”时，系统会在当天首次加载时扫描本地聊天记录，发现 `Agent Loop` 已在知识图谱中，因此把它加入“现在复习”。小陈不看笔记，先回答：“模型调用工具后，结果为什么还要作为 `role=tool` 消息回填？”他能说出大意但不够准确，于是在 WebUI 或对话中记录：

```text
我刚复习了 Agent Loop。请把 agent-loop 记录为回忆评分 3，
并告诉我下一次什么时候复习。
```

Whale 调用：

```json
{"action":"rate","concept_id":"agent-loop","rating":3}
```

工具返回 `due_on`，小陈下次进入时可问“今天有哪些到期复习”，对应调用是 `{"action":"due"}`。如果他评分 1 或 2，阶段会回到最短间隔；这不是惩罚，而是告诉路线模块要继续拆小练习。

## 25.7 验证

```bash
./.venv/bin/python -m pytest tests/test_learning.py tests/test_webui_settings.py -q
```

测试固定日期：第一次评分 5 后到期日为 3 天后；下一次评分 1 后重置为 1 天后，并在当天的到期列表中出现。另有测试写入本地会话 JSONL，确认已知概念会进入复习表、不会自动带上评分，WebUI API 能读取该表。

## 当前边界

- 没有系统通知或日历同步；自动检索发生在 WebUI 打开期间的每日检查，学习者也可在对话中主动询问“今天该复习什么”。
- 没有把成绩或学习记录上传给第三方。
- 间隔固定且保守，后续可在保留本地数据契约的前提下换成更细的算法。
- 聊天扫描是对已有概念的字面匹配，不理解同义词，也不把出现次数当作掌握度或回忆评分。
